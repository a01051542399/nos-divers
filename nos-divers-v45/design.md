# nos divers 다이빙 정산 & 면책동의서 앱 - 디자인 문서

## 앱 개요

nos divers 다이빙 팀의 투어 비용 정산과 면책동의서 전자서명을 간편하게 관리하는 모바일 앱.

---

## 스크린 목록

| 스크린 | 설명 |
|--------|------|
| **투어 목록 (홈)** | 생성된 투어 리스트, 각 투어의 정산 상태 요약 |
| **투어 생성** | 새 투어 이름, 날짜, 장소 입력 |
| **투어 상세** | 투어 정보, 참여자 목록, 비용 항목, 정산 결과 |
| **비용 항목 추가** | 비용 이름, 금액, 결제자 선택 |
| **정산 결과** | N분의1 계산 결과, 누가 누구에게 얼마 보내야 하는지 |
| **면책동의서 목록** | 투어별 면책동의서 서명 현황 |
| **면책동의서 서명** | 동의서 내용 표시 + 서명 패드 + 서명자 이름 입력 |
| **서명 완료 확인** | 서명된 동의서 미리보기 |
| **설정** | 앱 정보, 데이터 관리 |

---

## 탭 구조 (3탭)

1. **투어** (house.fill) - 투어 목록 및 정산 관리
2. **동의서** (doc.text.fill) - 면책동의서 서명 관리
3. **설정** (gearshape.fill) - 앱 설정

---

## 주요 콘텐츠 및 기능

### 투어 탭
- **투어 목록**: 카드 형태로 투어 나열 (투어명, 날짜, 장소, 참여자 수, 총 비용)
- **투어 생성 폼**: 투어명, 날짜(DatePicker), 장소 입력
- **투어 상세 화면**:
  - 투어 기본 정보 헤더
  - 참여자 섹션: 참여자 추가/삭제, 이름 입력
  - 비용 항목 섹션: 비용명, 금액, 결제자(누가 냈는지), N/1 대상자 선택
  - 정산 결과 섹션: 1인당 금액, 송금 안내 (A → B: 50,000원)

### 동의서 탭
- **투어별 동의서 현황**: 투어 선택 → 서명 현황 리스트
- **서명 화면**: 
  - 면책동의서 전문 표시 (스크롤)
  - 서명자 이름 입력
  - 캔버스 서명 패드 (손가락으로 서명)
  - 서명 완료 버튼
- **서명 확인**: 서명된 동의서 + 서명 이미지 + 날짜/시간

### 설정 탭
- 앱 버전 정보
- 데이터 초기화
- 면책동의서 템플릿 관리

---

## 핵심 사용자 플로우

### 플로우 1: 투어 정산
1. 홈 탭 → "새 투어 만들기" 버튼 탭
2. 투어명, 날짜, 장소 입력 → 생성
3. 투어 상세 → "참여자 추가" → 이름 입력
4. "비용 추가" → 비용명, 금액, 결제자 선택, N/1 대상자 선택
5. 자동 정산 결과 확인 (누가 누구에게 얼마)

### 플로우 2: 면책동의서 서명
1. 동의서 탭 → 투어 선택
2. "새 서명 받기" 탭
3. 서명자 이름 입력
4. 동의서 내용 확인 (스크롤)
5. 서명 패드에 서명
6. 서명 완료 → 서명 기록 저장

## 데이터 모델

### Tour (투어)
```
{
  id: string (UUID),
  name: string,
  date: string (ISO),
  location: string,
  participants: Participant[],
  expenses: Expense[],
  createdAt: string
}
```

### Participant (참여자)
```
{
  id: string (UUID),
  name: string
}
```

### Expense (비용 항목)
```
{
  id: string (UUID),
  name: string,
  amount: number,
  paidBy: string (participant id),
  splitAmong: string[] (participant ids)
}
```

### Waiver (면책동의서)
```
{
  id: string (UUID),
  tourId: string,
  signerName: string,
  signatureImage: string (base64),
  signedAt: string (ISO),
  agreed: boolean
}
```

## 레이아웃 원칙

- 모바일 세로 모드(9:16) 최적화, 한 손 사용 고려
- Apple HIG 준수: 큰 타이틀, 충분한 터치 타겟(44pt), 명확한 계층 구조
- 카드 기반 UI로 투어/비용 항목 표시
- 바텀 시트 활용하여 빠른 입력 (참여자 추가, 비용 추가)
- 정산 결과는 시각적으로 명확하게 (화살표 + 금액)
