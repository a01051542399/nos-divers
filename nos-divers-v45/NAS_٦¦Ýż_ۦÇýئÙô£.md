# NoS Divers 앱 - 시놀로지 NAS 설치 가이드

> **이 가이드는 컴퓨터 초보자를 위해 작성되었습니다.**
> 모든 단계를 천천히 따라하시면 됩니다. 모르는 용어가 나오면 바로 아래에 설명이 있습니다.

---

## 전체 흐름 한눈에 보기

설치는 크게 **3단계**입니다. 아래 표에서 현재 어디까지 했는지 체크하면서 진행하세요.

| 순서 | 할 일 | 예상 시간 | 난이도 |
|------|--------|-----------|--------|
| **1단계** | NAS에 Docker 설치하기 | 5분 | 쉬움 |
| **2단계** | 서버 파일 업로드하고 실행하기 | 15분 | 보통 |
| **3단계** | 외부에서 접속할 수 있게 설정하기 | 20분 | 보통 |
| **선택** | OneDrive 자동 업로드 설정 | 10분 | 쉬움 |

## 1단계: NAS에 Docker 설치하기

### Docker가 뭔가요?

Docker는 **"프로그램을 상자에 담아서 실행하는 도구"**라고 생각하시면 됩니다. 우리가 NAS에 서버를 설치할 때, 이 상자(컨테이너) 안에 서버를 넣어서 실행합니다. NAS의 다른 기능에 영향을 주지 않고 안전하게 돌릴 수 있습니다.

### 1-1. NAS에 로그인하기

컴퓨터에서 인터넷 브라우저(크롬, 사파리 등)를 열고, 주소창에 NAS의 IP 주소를 입력합니다.

```
http://192.168.xxx.xxx:5000
```

> **NAS IP 주소를 모르겠다면?**
> 시놀로지 홈페이지에서 [Synology Assistant](https://www.synology.com/ko-kr/support/download)를 다운로드하여 실행하면, 같은 네트워크에 있는 NAS를 자동으로 찾아줍니다.

로그인 화면이 나타나면 NAS 관리자 아이디와 비밀번호를 입력합니다.

### 1-2. Container Manager 설치하기

로그인 후 바탕화면에서 **패키지 센터** 아이콘을 클릭합니다. (앱스토어 같은 것입니다)

왼쪽 위 검색창에 **Container Manager**를 입력하고 검색합니다. (DSM 7.0~7.1 버전이면 **Docker**로 검색하세요)

검색 결과에서 **설치** 버튼을 누르고 기다립니다. 설치가 끝나면 **열기** 버튼이 나타납니다.

> **"Container Manager가 검색되지 않아요"** → NAS 모델이 Docker를 지원하지 않는 것입니다. DS220j, DS120j 같은 "j" 시리즈는 Docker를 사용할 수 없습니다. DS220+, DS720+, DS920+ 같은 "+" 시리즈가 필요합니다.

## 2단계: 서버 파일 업로드하고 실행하기

### 2-1. NAS에 폴더 만들기

바탕화면에서 **File Station** 아이콘을 클릭합니다. (윈도우의 파일 탐색기 같은 것입니다)

왼쪽 목록에서 **docker** 폴더를 찾아 클릭합니다.

> **docker 폴더가 없다면?**
> **제어판** → **공유 폴더** → **생성** 버튼을 눌러서 이름을 `docker`로 입력하고 만드세요.

`docker` 폴더 안에서 상단 메뉴의 **만들기** → **폴더 만들기**를 클릭하고, 이름을 `nos-divers`로 입력합니다.

이제 `nos-divers` 폴더 안에 들어가서, 같은 방법으로 아래 3개의 폴더를 추가로 만듭니다.

| 만들 폴더 이름 | 용도 |
|---------------|------|
| `mysql-data` | 데이터베이스 저장소 |
| `server` | 서버 프로그램 |
| `uploads` | PDF 파일 저장소 |

완성되면 이런 모양이 됩니다:

```
docker/
  └── nos-divers/
      ├── mysql-data/     (빈 폴더)
      ├── server/         (빈 폴더)
      └── uploads/        (빈 폴더)
```

### 2-2. ZIP 파일 업로드하기

제가 드린 **nos-divers-server-deploy.zip** 파일을 컴퓨터에 다운로드하고 압축을 풉니다.

압축을 풀면 이런 파일들이 나옵니다:

| 파일/폴더 | 어디에 업로드하나요? |
|-----------|---------------------|
| `docker-compose.yml` | `/docker/nos-divers/` 폴더에 |
| `init-database.sql` | `/docker/nos-divers/` 폴더에 |
| `server` 폴더 (안에 dist, drizzle, package.json) | `/docker/nos-divers/server/` 폴더에 **안의 내용물**을 |
| `uploads` 폴더 (안에 정산서, 면책동의서, 참여자명단) | `/docker/nos-divers/uploads/` 폴더에 **안의 내용물**을 |

**업로드하는 방법:**

File Station에서 `/docker/nos-divers/` 폴더로 이동한 후, 상단 메뉴에서 **업로드** → **업로드 - 덮어쓰기**를 클릭합니다. 그리고 `docker-compose.yml`과 `init-database.sql` 파일을 선택합니다.

다음으로 `/docker/nos-divers/server/` 폴더로 이동한 후, 압축 푼 폴더 안의 `server` 폴더 **안에 있는 파일들** (`dist` 폴더, `drizzle` 폴더, `package.json`)을 드래그해서 업로드합니다.

마지막으로 `/docker/nos-divers/uploads/` 폴더로 이동한 후, `uploads` 폴더 **안에 있는 하위 폴더들** (`정산서`, `면책동의서`, `참여자명단`)을 드래그해서 업로드합니다.

**업로드 완료 후 NAS 폴더 구조 확인:**

```
docker/
  └── nos-divers/
      ├── docker-compose.yml      ← 있어야 함
      ├── init-database.sql       ← 있어야 함
      ├── mysql-data/             ← 빈 폴더 (정상)
      ├── server/
      │   ├── dist/
      │   │   └── index.js        ← 있어야 함
      │   ├── drizzle/
      │   │   └── (여러 파일들)    ← 있어야 함
      │   └── package.json        ← 있어야 함
      └── uploads/
          ├── 정산서/
          ├── 면책동의서/
          └── 참여자명단/
```

### 2-3. 비밀번호 변경하기 (중요!)

`docker-compose.yml` 파일 안에 기본 비밀번호가 적혀 있습니다. **보안을 위해 반드시 변경해야 합니다.**

File Station에서 `docker-compose.yml` 파일을 **오른쪽 클릭** → **텍스트 편집기로 열기**를 선택합니다. (또는 컴퓨터에서 메모장으로 열어서 수정한 후 다시 업로드해도 됩니다)

파일 안에서 아래 **3곳**을 찾아서 본인만의 비밀번호로 변경합니다.

**변경할 곳 1 - MySQL 관리자 비밀번호 (14번째 줄 근처):**

```
변경 전: MYSQL_ROOT_PASSWORD: NosD1vers2026!db
변경 후: MYSQL_ROOT_PASSWORD: 본인이정한비밀번호1
```

**변경할 곳 2 - MySQL 앱 비밀번호 (17번째 줄 근처):**

```
변경 전: MYSQL_PASSWORD: NosD1vers2026!app
변경 후: MYSQL_PASSWORD: 본인이정한비밀번호2
```

**변경할 곳 3 - DATABASE_URL 안의 비밀번호 (30번째 줄 근처):**

이 줄에서 `NosD1vers2026!app` 부분을 **변경할 곳 2**와 **똑같은 비밀번호**로 바꿉니다.

```
변경 전: DATABASE_URL: mysql://nosdivers:NosD1vers2026!app@mysql:3306/nos_divers
변경 후: DATABASE_URL: mysql://nosdivers:본인이정한비밀번호2@mysql:3306/nos_divers
```

**변경할 곳 4 - JWT 시크릿 (32번째 줄 근처):**

```
변경 전: JWT_SECRET: NosD1vers_JWT_Secret_2026_Change_This!
변경 후: JWT_SECRET: 아무거나길게적으세요예를들면NosDivers자이언트2026
```

> **비밀번호 팁**: 영문, 숫자를 섞어서 12자 이상으로 만드세요. 예: `MyDiving2026pass`

변경이 끝나면 파일을 **저장**합니다.

### 2-4. Docker 실행하기

**Container Manager**를 엽니다. (바탕화면 아이콘 또는 메인 메뉴에서)

왼쪽 메뉴에서 **프로젝트**를 클릭합니다.

오른쪽 위의 **생성** 버튼을 클릭합니다.

아래와 같이 설정합니다:

| 설정 항목 | 입력할 값 |
|-----------|-----------|
| 프로젝트 이름 | `nos-divers` |
| 경로 | **찾아보기** 클릭 → `docker` → `nos-divers` 폴더 선택 |
| 소스 | `기존 docker-compose.yml 사용` 선택 |

**다음** 버튼을 클릭하면 docker-compose.yml 내용이 표시됩니다. 내용을 확인하고 **다음** → **완료**를 클릭합니다.

잠시 기다리면 Docker가 필요한 프로그램들을 인터넷에서 자동으로 다운로드하고 설치합니다. (처음에는 5~10분 정도 걸릴 수 있습니다)

### 2-5. 실행 확인하기

Container Manager의 **프로젝트** 메뉴에서 `nos-divers`를 클릭하면 **컨테이너** 탭에서 2개의 항목이 보입니다.

| 이름 | 상태 | 의미 |
|------|------|------|
| nos-divers-db | **실행 중** (초록색) | 데이터베이스가 정상 작동 중 |
| nos-divers-server | **실행 중** (초록색) | 서버가 정상 작동 중 |

둘 다 **초록색 "실행 중"**이면 성공입니다!

> **"nos-divers-server가 빨간색이에요"** → 데이터베이스가 아직 준비되지 않은 것일 수 있습니다. 1~2분 기다린 후 **nos-divers-server**를 선택하고 상단의 **시작** 버튼을 눌러보세요.

### 2-6. 데이터베이스 테이블 만들기

서버가 실행되었지만, 아직 데이터를 저장할 **테이블**(엑셀의 시트 같은 것)이 없습니다. 이것을 만들어야 합니다.

**이 단계는 SSH라는 것을 사용합니다. 처음이시면 아래를 천천히 따라하세요.**

**SSH가 뭔가요?** NAS에 글자로 명령을 내리는 방법입니다. 마우스 대신 키보드로 NAS를 조작한다고 생각하시면 됩니다.

---

**SSH 준비하기 (한 번만 하면 됩니다):**

NAS DSM에서 **제어판**을 엽니다. **터미널 및 SNMP**를 클릭합니다. **SSH 서비스 활성화**에 체크하고 **적용**을 누릅니다.

---

**SSH 접속하기:**

**윈도우 사용자:**

키보드에서 `윈도우키 + R`을 누르고, `cmd`를 입력한 후 엔터를 칩니다. 검은 화면(명령 프롬프트)이 열립니다. 아래 명령어를 입력합니다. (NAS의 IP주소를 본인 것으로 바꾸세요)

```
ssh admin@192.168.xxx.xxx
```

**맥 사용자:**

`Command + Space`를 누르고 `터미널`을 입력하여 터미널을 엽니다. 아래 명령어를 입력합니다.

```
ssh admin@192.168.xxx.xxx
```

---

처음 접속하면 `Are you sure you want to continue connecting?`이라고 물어봅니다. **yes**를 입력하고 엔터를 칩니다.

비밀번호를 입력하라고 나옵니다. **NAS 관리자 비밀번호**를 입력합니다. (입력할 때 화면에 글자가 안 보이는 게 정상입니다. 그냥 입력하고 엔터를 치세요)

접속에 성공하면 `admin@NAS이름:~$` 같은 글자가 나타납니다.

---

**테이블 만들기 명령어 실행:**

아래 명령어를 **한 줄씩** 복사해서 붙여넣고 엔터를 칩니다.

> **주의**: 아래 명령어에서 `본인이정한비밀번호2` 부분을 2-3단계에서 설정한 **MYSQL_PASSWORD** 비밀번호로 바꾸세요. `-p` 뒤에 **띄어쓰기 없이** 바로 비밀번호를 붙여 쓰세요.

```
sudo docker exec -i nos-divers-db mysql -u nosdivers -p본인이정한비밀번호2 nos_divers < /volume1/docker/nos-divers/init-database.sql
```

예를 들어 비밀번호가 `MyDiving2026pass`이면:

```
sudo docker exec -i nos-divers-db mysql -u nosdivers -pMyDiving2026pass nos_divers < /volume1/docker/nos-divers/init-database.sql
```

실행 후 아래와 비슷한 결과가 나오면 성공입니다:

```
result
✅ 모든 테이블이 성공적으로 생성되었습니다!
Tables_in_nos_divers
expenses
participants
tours
users
waivers
```

> **에러가 나왔다면?** `Access denied`라고 나오면 비밀번호가 틀린 것입니다. docker-compose.yml에서 설정한 MYSQL_PASSWORD를 다시 확인하세요.

SSH 작업이 끝났으면 `exit`를 입력하고 엔터를 쳐서 접속을 종료합니다.

### 2-7. 서버 접속 테스트

컴퓨터에서 인터넷 브라우저를 열고 주소창에 아래를 입력합니다:

```
http://NAS의_IP주소:3000
```

예를 들어 NAS IP가 192.168.1.100이면: `http://192.168.1.100:3000`

화면에 뭔가 응답이 오면 (빈 페이지여도 괜찮습니다) **서버 설치 성공**입니다!

> **"사이트에 연결할 수 없습니다"가 나온다면?** Container Manager에서 nos-divers-server가 실행 중인지 확인하세요. 실행 중이 아니면 선택 후 **시작** 버튼을 누르세요.

## 3단계: 외부에서 접속할 수 있게 설정하기

지금까지는 **같은 Wi-Fi**에 연결된 상태에서만 서버에 접속할 수 있습니다. 밖에서 LTE/5G로도 접속하려면 추가 설정이 필요합니다.

### 이 단계가 왜 필요한가요?

집 안의 NAS는 공유기(인터넷 공유기) 뒤에 숨어있습니다. 밖에서 오는 요청은 공유기가 막아버립니다. 그래서 **"이 요청은 NAS로 보내줘"**라고 공유기에게 알려줘야 합니다. 이것을 **포트포워딩**이라고 합니다.

또한 집의 인터넷 주소(IP)는 수시로 바뀝니다. 바뀔 때마다 앱 설정을 변경할 수 없으니, **고정된 이름**(예: nosdivers.synology.me)을 만들어서 항상 같은 이름으로 접속할 수 있게 합니다. 이것을 **DDNS**라고 합니다.

### 3-1. DDNS 설정하기 (고정 주소 만들기)

NAS DSM에서 **제어판**을 엽니다.

**외부 액세스**를 클릭합니다.

상단 탭에서 **DDNS**를 클릭합니다.

**추가** 버튼을 클릭합니다.

아래와 같이 설정합니다:

| 설정 항목 | 입력할 값 |
|-----------|-----------|
| 서비스 공급자 | **Synology** 선택 |
| 호스트 이름 | 원하는 이름 입력 (예: `nosdivers`) |

호스트 이름을 `nosdivers`로 입력하면, 앞으로 **nosdivers.synology.me**라는 주소로 NAS에 접속할 수 있습니다.

**연결 테스트** 버튼을 눌러서 **"정상"**이라고 나오면 **확인**을 클릭합니다.

> **"이미 사용 중인 호스트 이름입니다"** → 다른 사람이 이미 그 이름을 쓰고 있습니다. 다른 이름을 시도하세요. (예: `nosdivers-giant`, `nosdiversteam` 등)

### 3-2. 포트포워딩 설정하기 (공유기에서)

이제 **공유기** 설정을 변경해야 합니다. 공유기 종류에 따라 화면이 다르지만, 하는 일은 같습니다.

**공유기 관리 페이지 접속하기:**

인터넷 브라우저에서 아래 주소 중 하나를 입력합니다. (공유기마다 다릅니다)

| 공유기 종류 | 관리 페이지 주소 | 기본 아이디/비밀번호 |
|------------|-----------------|---------------------|
| **ipTIME** | `192.168.0.1` | admin / admin |
| **KT 공유기** | `172.30.1.254` 또는 `192.168.1.1` | ktuser / homehub |
| **SK 공유기** | `192.168.25.1` | admin / admin |
| **LG U+ 공유기** | `192.168.219.1` | admin / admin |

## (선택) OneDrive 자동 업로드 설정하기

정산서 PDF, 면책동의서 PDF 파일을 OneDrive에 자동으로 올리고 싶다면, 시놀로지의 **Cloud Sync** 기능을 사용합니다.

### Cloud Sync가 뭔가요?

NAS의 특정 폴더와 OneDrive 폴더를 **자동으로 연결**해주는 기능입니다. NAS 폴더에 파일이 생기면 자동으로 OneDrive에도 올라갑니다.

### 설정 방법

**패키지 센터**에서 **Cloud Sync**를 검색하여 설치합니다.

Cloud Sync를 열고 왼쪽 아래 **+** 버튼을 클릭합니다.

클라우드 서비스 목록에서 **Microsoft OneDrive**를 선택하고 **다음**을 클릭합니다.

Microsoft 로그인 화면이 나타나면 **Microsoft 계정으로 로그인**하고 권한을 허용합니다.

동기화 설정 화면에서 아래와 같이 입력합니다:

| 설정 항목 | 입력할 값 |
|-----------|-----------|
| 로컬 경로 | `/docker/nos-divers/uploads` 선택 |
| 원격 경로 | OneDrive에서 원하는 폴더 선택 (예: `NoS Divers`) |
| 동기화 방향 | **로컬 변경사항만 업로드** 선택 |

**다음** → **완료**를 클릭합니다.

이제 서버가 `/uploads/정산서/`, `/uploads/면책동의서/`, `/uploads/참여자명단/` 폴더에 PDF를 저장하면, Cloud Sync가 자동으로 OneDrive에 업로드합니다.

## 백업 설정 (데이터 안전하게 보관하기)

투어 데이터, 면책동의서 등 중요한 데이터를 매일 자동으로 백업하는 설정입니다.

### 자동 백업 설정하기

NAS DSM에서 **제어판** → **작업 스케줄러**를 엽니다.

**생성** → **예약된 작업** → **사용자 정의 스크립트**를 클릭합니다.

**일반** 탭에서:

| 설정 항목 | 입력할 값 |
|-----------|-----------|
| 작업 이름 | `NoS Divers DB 백업` |
| 사용자 | `root` |

**일정** 탭에서:

| 설정 항목 | 입력할 값 |
|-----------|-----------|
| 실행 빈도 | 매일 |
| 시간 | 03:00 (새벽 3시) |

**작업 설정** 탭에서 아래 스크립트를 **그대로 복사**해서 붙여넣습니다:

> **주의**: 아래에서 `본인이정한비밀번호2` 부분을 docker-compose.yml에서 설정한 MYSQL_PASSWORD로 바꾸세요.

```bash
#!/bin/bash
BACKUP_DIR="/volume1/docker/nos-divers/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
docker exec nos-divers-db mysqldump -u nosdivers -p본인이정한비밀번호2 nos_divers > $BACKUP_DIR/backup_$DATE.sql
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
```

**확인**을 클릭합니다.

이제 매일 새벽 3시에 데이터베이스가 자동으로 백업됩니다. 30일이 지난 오래된 백업은 자동으로 삭제됩니다.

## 문제가 생겼을 때 (FAQ)

### "Container Manager에서 프로젝트가 시작되지 않아요"

docker-compose.yml 파일에 오타가 있을 수 있습니다. 특히 **들여쓰기**(앞의 빈칸)가 중요합니다. 탭(Tab) 대신 **스페이스 2칸**을 사용해야 합니다. 가장 확실한 방법은 제가 드린 원본 파일을 다시 업로드하고, 비밀번호만 변경하는 것입니다.

### "nos-divers-server가 계속 재시작돼요"

Container Manager에서 nos-divers-server를 클릭하고 **로그** 탭을 확인하세요. 에러 메시지가 보이면 저에게 알려주시면 해결해 드리겠습니다. 가장 흔한 원인은 데이터베이스 비밀번호가 일치하지 않는 것입니다.

### "SSH 접속이 안 돼요"

NAS DSM에서 **제어판** → **터미널 및 SNMP**에서 **SSH 서비스 활성화**가 체크되어 있는지 확인하세요. 포트 번호가 22가 아닌 다른 번호로 되어 있다면, `ssh admin@NAS_IP -p 포트번호`로 접속하세요.

### "외부에서 접속이 안 돼요"

같은 Wi-Fi에서는 되는데 밖에서만 안 되는 경우, 공유기 포트포워딩 설정을 다시 확인하세요. 특히 **내부 IP 주소**가 NAS의 실제 IP와 일치하는지 확인합니다. 그래도 안 되면 인터넷 서비스 업체에서 포트를 차단하고 있을 수 있으므로, Cloudflare Tunnel 사용을 권장합니다.

### "NAS를 재부팅하면 서버가 자동으로 켜지나요?"

네, docker-compose.yml에 `restart: always` 설정이 되어 있어서 NAS가 재부팅되면 서버도 자동으로 시작됩니다. 단, Container Manager(Docker) 패키지가 자동 시작으로 설정되어 있어야 합니다. **패키지 센터** → **설치됨** → **Container Manager** 에서 확인하세요.
